# DOE Project Repository Template

Use this checklist when creating a new DOE project repository.

## Repository Name

```text
doe-<project-short-name>
```

## Required Files

- `README.md`
- `LICENSE`
- `CONTRIBUTING.md`
- `SECURITY.md`
- `CODEOWNERS`
- `.github/ISSUE_TEMPLATE/deliverable.md`
- `.github/pull_request_template.md`

## Required Folders

```text
docs/
deliverables/
reports/
presentations/
publications/
src/
data/
results/
meetings/
```

## README Sections

Every project README should include:

- Project title
- DOE award or project identifier, if shareable
- Principal investigator and project leads
- Repository purpose
- Folder structure
- Deliverable list
- Publication list
- Data handling notes
- Access and contribution rules

## Deliverable Tracking

Each deliverable should include:

- Deliverable ID
- Title
- Due date
- Responsible person
- Status
- Link to final artifact
- Sponsor submission date, if applicable
