# Contributing

## Branches

Use short, descriptive branch names:

```text
feature/<short-description>
fix/<short-description>
docs/<short-description>
deliverable/<deliverable-id>
```

## Pull Requests

Use pull requests for changes to:

- source code
- reports
- deliverables
- published data
- project documentation
- repository policies

At least one project lead or repository maintainer should review important changes before merge.

## Commit Messages

Use concise commit messages that explain the change:

```text
Add Q2 deliverable draft
Update experiment result summary
Fix simulation configuration parser
```

## Sensitive Information

Do not commit:

- passwords
- API keys
- private tokens
- unpublished sponsor-sensitive reports unless the repository is private and access is approved
- controlled or restricted data without explicit approval
