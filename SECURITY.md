# Security Policy

## Sensitive Research Content

DOE project repositories may contain sensitive project information. Keep active project repositories private unless the project lead has approved public release.

Do not commit credentials, tokens, private keys, passwords, or production system details.

## Reporting Security Issues

Report security concerns to the project lead or repository maintainer. If the issue affects a DOE-funded deliverable, notify the principal investigator before public disclosure.

## Access Reviews

Review repository access at least once per semester and when a student, collaborator, or staff member leaves the project.
